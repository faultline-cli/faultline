# Faultline v0.4.1

Faultline v0.4.1 is a corpus-expansion and coverage-hardening maintenance
release. The playbook catalog is unchanged; the accepted real fixture corpus
grows from 174 to 228 fixtures, and Go test coverage is now tracked and
reported in CI.

## What This Release Delivers

- **170 bundled playbooks** (unchanged from v0.4.0).
- **228 accepted real fixtures**, up 54 from v0.4.0. New fixtures span
  artifact upload failures, connection-refused variants across MySQL, Redis,
  and Postgres, docker-daemon-unavailable, env-var-missing, pip-install-failure,
  permission-denied, runtime-mismatch, and additional coverage across
  GitHub, GitLab, Stack Exchange, Reddit, and Discourse sources.
- **Top-1 accuracy 1.000** maintained with zero unmatched results and zero
  false positives across the expanded corpus.
- **Go test coverage 79.7%** — collected with `-coverprofile` on every CI run
  and uploaded to Codecov. Coverage badge added to README.
- **CI hardening** — the `validate` job now runs `go test ./... -coverprofile`
  directly, then uploads to Codecov. The `make review`, fixture-check, and
  cli-smoke gates are unchanged.

## Release Boundary Status

No change to the default shipping narrative. The default user story remains:

- `analyze`
- `workflow`
- `list`
- `explain`
- `fix`

Companion and maintainer surfaces remain supported but non-default:
`inspect`, `guard`, `trace`, `replay`, `compare`, `packs`, and hidden
`fixtures` workflows.

## Remaining Gaps

- **Corpus mix is still GitHub-heavy.** 116 of 228 accepted fixtures are
  GitHub (51%); future ingestion should keep biasing toward GitLab, Stack
  Exchange, Reddit, and Discourse.
- **Weak-match rate 0.132 (30/228).** Targeted `none:` exclusion fixes
  can reduce this further in the next pass.
- **Bayes remains opt-in.** The known Bayes regression on
  `gitlab-gitlab-org-gitlab-runner-6557-s3-64c99cfe7a2f9dfa` is still
  unresolved, so `--bayes` stays outside the default release path.
- **Go coverage 79.7%.** Tool and eval-corpus packages bring the average
  down; core engine and workflow packages are uniformly above 83%.
