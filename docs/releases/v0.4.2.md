# Faultline v0.4.2

Faultline v0.4.2 is a playbook-quality and test-coverage maintenance release.
The accepted real fixture corpus is unchanged at 228 fixtures; three new
playbooks land, 12 existing playbooks gain additional pattern coverage, and
the weak-match rate drops from 0.132 to 0.009. Go test coverage rises to
81.7%.

## What This Release Delivers

- **181 bundled playbooks** (up 3 from v0.4.1's 178).
- **3 new playbooks**: `junit-test-failure`, `link-checker-failure`, and
  `test-assertion-with-reason`.
- **12 playbooks with expanded pattern coverage**: `git-auth`, `ssh-key-auth`,
  `gradle-build`, `syntax-error`, `network-timeout`, `ssl-cert-error`,
  `port-in-use`, `segfault`, `artifact-missing`, `clock-drift`,
  `go-test-failure`, `snapshot-mismatch`, and `test-timeout`.
- **Weak-match rate 0.009** — down from 0.132 in v0.4.1. Only 2 of 228
  accepted real fixtures produce a weak match; all others are strong.
- **Top-1 accuracy 1.000** maintained across the full 228-fixture real corpus.
- **Go test coverage 81.7%** — up from 79.7% in v0.4.1. Coverage gains are
  concentrated in `tools/eval-corpus` (compare: 43→96.5%, gaps: 53→97.0%,
  report: 72→87.6%, manifest: 70→88.6%) and `internal/store` (73→74.1%).
- **3 new ADRs** documenting silent-failure detection, playbook catalog
  scalability, and the CI failure ontology (ADR-0007, ADR-0008, ADR-0009).
- **Updated shipped examples** for `missing-executable` and
  `missing-vs-runtime` to match current engine output.

## Release Boundary Status

No change to the default shipping narrative. The default user story remains:

- `analyze`
- `workflow`
- `list`
- `explain`
- `fix`

Companion and maintainer surfaces remain supported but non-default:
`inspect`, `guard`, `trace`, `replay`, `compare`, `packs`, and hidden
`fixtures` workflows.

## Remaining Gaps

- **Corpus mix is still GitHub-heavy.** 116 of 228 accepted fixtures are
  GitHub (51%); future ingestion should keep biasing toward GitLab, Stack
  Exchange, Reddit, and Discourse.
- **2 weak-match fixtures remaining.** Targeted `none:` exclusion fixes
  could close these in the next pass.
- **Bayes remains opt-in.** The known Bayes regression on
  `gitlab-gitlab-org-gitlab-runner-6557-s3-64c99cfe7a2f9dfa` is still
  unresolved, so `--bayes` stays outside the default release path.
- **`internal/app` coverage 52.0%.** High-integration app-layer functions
  (`WorkflowExplain`, `WorkflowApply`, etc.) require end-to-end wiring to
  test; deferred.
- **`internal/cli` coverage 60.1%.** CLI integration paths need an
  invocation harness; deferred.
