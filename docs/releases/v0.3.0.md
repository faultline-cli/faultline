# Faultline v0.3.0

Faultline v0.3.0 hardens the deterministic CLI release boundary by extending checked-in validation to companion command surfaces while keeping the default user story focused on the local `analyze` -> `workflow` path. GitHub Actions remains a strong follow-up integration path built on the same stable CLI artifacts.

## What This Release Delivers

- Expanded deterministic CLI smoke coverage across companion commands: `trace`, `replay`, `compare`, `inspect`, and `guard`
- Checked-in replay, trace, and compare snapshots in `examples/` and smoke validation against those snapshots
- Deterministic source-repo smoke setup for `inspect`/`guard` behavior (including guard non-zero contract on findings)
- README and release-facing docs aligned to the shipped v0.3.0 workflow, pinned install examples, and local-first default path

## Release Boundary Status

The default shipping narrative remains centered on:

- `analyze`
- `workflow`
- `list`
- `explain`
- `fix`

Companion surfaces remain supported and fully validated, but non-default:

- `trace`, `replay`, `compare`, `inspect`, `guard`, `packs`

Experimental provider-backed delta remains opt-in via:

- `FAULTLINE_EXPERIMENTAL_GITHUB_DELTA=1`
