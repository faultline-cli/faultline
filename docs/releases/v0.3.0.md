# Faultline v0.3.0 Graduation Path

> This roadmap frames the v0.3.0 release as a "Graduation Path" where experimental surfaces promote from "Complete" to "Stable" based on validation and adoption thresholds.

## Core Theme

v0.3.0 is about proving the latent product identity: Faultline as an agent-compatible deterministic substrate. This release promotes `trace`, `replay`, `compare`, `inspect`, and `guard` from "Complete" (supported but not-default) to "Stable" (first-class surfaces with equivalent release validation).

## v0.3.0 Feature Highlights

- **Tier 2 promotion**: `trace`, `replay`, `compare`, `inspect`, `guard` promoted from Complete → Stable
- **Expanded playbook coverage**: 77 → 90+ bundled playbooks
- **Expanded fixture corpus**: 103 → 120+ real-fixture regression proofs
- **Reduced weak match rate**: 0.107 → < 0.08
- **CLI smoke expansion**: 5 → 10 validated surfaces

## Graduation Path

### Tier 1: Stable (Ship-Ready Default)

These commands form the v0.3.0 default release story:

| Command | Status | Validation Required |
| --- | --- | --- |
| `analyze` | Stable | Core: fixture regression, JSON contract stability |
| `workflow` | Stable | Agent JSON contract, local text output |
| `list` | Stable | Catalog enumeration |
| `explain` | Stable | Playbook rendering |

**v0.3.0 goal**: Ship `analyze` → `workflow` as the unchanging default experience.

### Tier 2: Promotion to Stable

These commands promote from "Complete" (tested, supported, non-default) to "Stable" (first-class surfaces):

| Command | Status | Promotion Criteria |
| --- | --- | --- |
| `trace` | Complete → Stable | Default help surface, core docs integration |
| `replay` | Complete → Stable | Release checks, CLI smoke validation |
| `compare` | Complete → Stable | Diff semantics, example comparisons |
| `inspect` | Complete → Stable | Source-level scope, bounded checks |
| `guard` | Complete → Stable | Quiet-mode contract, high-confidence threshold |

**Promotion contract**:

- [ ] Deterministic command coverage (parsing, error handling, edge cases)
- [ ] Fixture-backed regression proof (new fixtures for each command's behavior)
- [ ] Checked-in example snapshots in `examples/`
- [ ] CLI smoke integration (`make cli-smoke` includes new surfaces)
- [ ] Help surface integration (visible in `faultline --help`)
- [ ] Core docs page integration

### Tier 3: Experimental (Opt-In Only)

These remain experimental and require opt-in:

| Command | Status | Next Steps |
| --- | --- | --- |
| `--delta-provider github-actions` | Experimental | Gate behind `FAULTLINE_EXPERIMENTAL_GITHUB_DELTA=1` until coverage matches core |

### Tier 4: Deferred

These capabilities exist but stay out of the v0.3.0 narrative:

- Broad "CI automation platform" framing
- Provider-backed delta as default product path
- Pack management as onboarding surface

## Release Validation

v0.3.0 is release-ready when:

```bash
make test              # All unit tests pass
make review           # Playbook overlap review passes
make fixture-check   # Fixture regression passes (103 baseline)
make cli-smoke      # All 10 surfaces validated
make release-check VERSION=v0.3.0  # Archive and Docker packaging pass
```

## Adoption Signals

Graduate from "Complete" to "Stable" in the next release (v0.4.0) when:

- Adoption threshold: > 500 installs per month (measured via install.sh or archive downloads)
- Usage signal: Non-default command usage > 10% of total invocations
- Documentation signal: Community-contributed examples for promoted surfaces
- Support signal: < 5% issue rate on promoted surfaces

These thresholds ensure surfaces are battle-tested before becoming part of the default story.
