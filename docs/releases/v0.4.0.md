# Faultline v0.4.0

Faultline v0.4.0 strengthens the shipped deterministic CLI around its default
user path: `analyze`, `workflow`, `list`, `explain`, and `fix`. The release
focuses on higher-value bundled playbook coverage, stronger fixture and source
regression proof, and a cleaner real-corpus trust boundary.

## What This Release Delivers

- **100 bundled playbooks** in the shipped starter catalog, including ten
  premium ports for common CI and delivery failures:
  `gradle-build`, `maven-dependency-resolution`, `dotnet-restore`,
  `terraform-init`, `terraform-state-lock`, `kubectl-auth`,
  `oidc-token-failure`, `testcontainer-startup`, `pytest-fixture-error`, and
  `jest-worker-crash`.
- **Real-corpus hardening** to 118 accepted fixtures with zero unmatched
  results, zero false positives, and zero weak matches in the checked baseline.
- **Targeted test-surface expansion** with accepted real fixtures for
  `pytest-fixture-error` and two distinct `testcontainer-startup` forms,
  including a GitLab DinD/Testcontainers failure and a Stack Overflow Maven
  integration-test case.
- **Adapter-diversity refinement** with an accepted GitLab `pipeline-timeout`
  fixture and a Reddit `missing-executable` fixture, plus a tighter
  `gitlab-ci-yaml-invalid` boundary so `.gitlab-ci.yml` filenames no longer
  outrank real timeout evidence.
- **Targeted catalog refinement** across the existing core playbooks:
  Docker auth vs image-pull overlap, artifact upload vs generic exit-code
  noise, GitHub Actions validation vs malformed action refs, CircleCI config
  validation coverage, DNS vs traceback noise, and source-specific weak-match
  leaks in the fixture corpus.
- **Dedicated source-playbook regression fixtures** for
  `missing-error-propagation` and `panic-in-http-handler`, including positive
  and mitigated repository examples with deterministic engine tests.
- **Fixture and authoring workflow polish** documenting source-detector
  regression fixtures as a first-class maintainer path instead of forcing that
  evidence into the real log corpus.

## Release Boundary Status

No change to the default shipping narrative. The default user story remains:

- `analyze`
- `workflow`
- `list`
- `explain`
- `fix`

Companion and maintainer surfaces remain supported but non-default:
`inspect`, `guard`, `trace`, `replay`, `compare`, `packs`, and hidden
`fixtures` workflows.

## Remaining Gaps

The release is materially stronger, but three follow-on improvements are still
worth calling out:

- **Corpus mix is still GitHub-heavy.** The accepted real corpus is 81 GitHub
  fixtures out of 118 total, so future ingestion should keep biasing toward
  GitLab, Discourse, Reddit, and Stack Exchange when strong evidence is
  available.
- **Bayes remains opt-in.** The known Bayes regression on
  `gitlab-gitlab-org-gitlab-runner-6557-s3-64c99cfe7a2f9dfa` is still not
  resolved, so `--bayes` stays outside the default release path.
- **Premium source playbooks are still deferred.** The source-detector surface
  is now better regression-gated, but deeper premium source coverage should
  land only with equivalent repository fixtures and companion-surface proof for
  `inspect` and `guard`.
