# Faultline v0.4.0

Faultline v0.4.0 strengthens the shipped deterministic CLI around its default
user path: `analyze`, `workflow`, `list`, `explain`, and `fix`. The release
focuses on higher-value bundled playbook coverage, stronger fixture and source
regression proof, and a cleaner real-corpus trust boundary.

## What This Release Delivers

- **123 bundled playbooks** in the shipped starter catalog, up from 77 in v0.3.0,
  including coverage for `gradle-build`, `maven-dependency-resolution`,
  `dotnet-restore`, `terraform-init`, `terraform-state-lock`, `kubectl-auth`,
  `oidc-token-failure`, `testcontainer-startup`, `pytest-fixture-error`, and
  `jest-worker-crash`.
- **Real-corpus hardening** to 174 accepted fixtures with top-1 accuracy 1.000,
  zero unmatched results, and zero false positives.
- **Weak-match rate reduced to 0.057** (10/174), down from 0.115 in v0.3.1, via
  targeted `none:` exclusion fixes across five playbooks:
  `pnpm-lockfile-missing` no longer fires on `ERR_PNPM_OUTDATED_LOCKFILE` logs;
  `docker-permission-denied-nonroot` excludes SSH publickey auth failures;
  `github-actions-missing-checkout` excludes logs that show `uses: actions/checkout`;
  `dns-enotfound` excludes Python `ModuleNotFoundError` false positives;
  `dockerfile-copy-source-missing` excludes gcc compiler failures.
- **Top-1 rate 1.000** after correcting `reddit-r-devops-18gy2p2` fixture whose
  expected playbook (`ssh-key-auth`) explicitly excluded its log pattern —
  corrected to `ssh-permission-denied`.
- **Adapter-diversity refinement** with accepted fixtures across GitHub, GitLab,
  Reddit, Discourse, and Stack Exchange providers.
- **Fixture and authoring workflow polish** documenting source-detector
  regression fixtures as a first-class maintainer path.

## Release Boundary Status

No change to the default shipping narrative. The default user story remains:

- `analyze`
- `workflow`
- `list`
- `explain`
- `fix`

Companion and maintainer surfaces remain supported but non-default:
`inspect`, `guard`, `trace`, `replay`, `compare`, `packs`, and hidden
`fixtures` workflows.

## Remaining Gaps

The release is materially stronger, but three follow-on improvements are still
worth calling out:

- **Corpus mix is still GitHub-heavy.** The accepted real corpus is 98 GitHub
  fixtures out of 174 total, so future ingestion should keep biasing toward
  GitLab, Discourse, Reddit, and Stack Exchange when strong evidence is
  available.
- **Bayes remains opt-in.** The known Bayes regression on
  `gitlab-gitlab-org-gitlab-runner-6557-s3-64c99cfe7a2f9dfa` is still not
  resolved, so `--bayes` stays outside the default release path.
- **Premium source playbooks are still deferred.** The source-detector surface
  is now better regression-gated, but deeper premium source coverage should
  land only with equivalent repository fixtures and companion-surface proof for
  `inspect` and `guard`.
