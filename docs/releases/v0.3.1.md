# Faultline v0.3.1

Faultline v0.3.1 extends the maintainer fixture tooling with a sanitizer for safe corpus contributions, a Bayes comparison gate for evaluating scoring mode changes, and supporting documentation for the promotion workflow.

## What This Release Delivers

- **`faultline fixtures sanitize`** — deterministic masking of credential and identity patterns in staging fixtures before corpus promotion. Covers GitHub tokens, AWS keys, Authorization headers, URL credentials, JWTs, PEM keys, and email addresses. Scope and limitations documented in `docs/fixture-corpus.md`.
- **`faultline fixtures compare-modes`** — compares baseline vs Bayesian evaluation reports across the fixture corpus. Produces per-fixture rank deltas, aggregate improved/regressed/unchanged counts, and top-1/top-3 accuracy deltas.
- **`make bayes-check`** — informational gate that runs `fixtures compare-modes` against the real corpus. Does not block release by default; `--fail-on-regression` is available for CI gates.
- **Bayes Promotion Gate** documented in `docs/release-boundary.md` — defines what must hold before `--bayes` graduates to a default-on mode.
- **Sanitize contribution flow** integrated into `CONTRIBUTING.md` and `docs/fixture-corpus.md` with command examples and manual review guidance.
